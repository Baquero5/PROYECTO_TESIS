1. Introducción

SmartInventory AI es un sistema de gestión de inventario que integra inteligencia artificial para predecir la demanda de productos. El objetivo principal es ayudar a las empresas a tomar mejores decisiones de compra y reabastecimiento, reduciendo tanto el sobreinventario como los desabastecimientos.

Objetivo del Proyecto: Desarrollar un sistema que combine la gestión tradicional de inventario con modelos de machine learning para predecir la demanda futura de productos, optimizando así la cadena de suministro.

¿Por qué Inteligencia Artificial?

  • Exactitud: Los modelos ML pueden capturar patrones complejos que los métodos estadísticos tradicionales no detectan.
  • Automatización: Una vez entrenado, el modelo genera predicciones automáticamente sin intervención humana.
  • Escalabilidad: Puede predecir la demanda de miles de productos simultáneamente.
  • Adaptabilidad: Los modelos se reentrenan con nuevos datos para mantenerse actualizados.

5. Modelos de Machine Learning

5.1 ¿Por qué se eligieron estos modelos?

Se seleccionaron XGBoost y LightGBM por las siguientes razones:

  • Alto rendimiento: Ambos modelos están en el top de competiciones de Kaggle para problemas de regresión.
  • Velocidad: LightGBM es especialmente rápido con grandes volúmenes de datos.
  • Manejo de datos faltantes: Ambos manejan naturalmente valores nulos.
  • Regularización: Incluyen técnicas para evitar overfitting.
  • Comunidad: Amplio soporte documental y de comunidad.

5.2 XGBoost (Extreme Gradient Boosting)

¿Qué es? XGBoost es un algoritmo de ensemble learning que combina múltiples árboles de decisión de forma secuencial. Cada nuevo árbol corrige los errores del anterior.

  • Tipo: Gradient Boosting
  • Fortalezas: Alta precisión, manejo de features, regularización L1/L2
  • Debilidades: Más lento que LightGBM, mayor consumo de memoria
  • Uso en nuestro proyecto: Modelo base para comparar rendimiento

5.3 LightGBM (Light Gradient Boosting Machine)

¿Qué es? LightGBM es una versión optimizada de Gradient Boosting desarrollada por Microsoft. Utiliza técnicas de muestreo basado en gradientes y división de hojas por lado.

  • Tipo: Gradient Boosting optimizado
  • Fortalezas: Velocidad de entrenamiento, menor uso de memoria, buena precisión
  • Debilidades: Puede overfitting con pocos datos
  • Uso en nuestro proyecto: Modelo ganador por mejor R² y velocidad

5.5 ¿Por qué ganó LightGBM?

Aunque XGBoost fue más rápido, LightGBM fue seleccionado como modelo principal por:

  1. Mejor R² (0.744 vs 0.722): Explica mejor la varianza en los datos.
  2. Menor RMSE (3.502 vs 3.650): Menor error cuadrático medio.
  3. Menor tamaño (5.6 MB vs 13.3 MB): Más eficiente en almacenamiento.

6. Métricas de Evaluación

6.1 MAE (Mean Absolute Error)

Fórmula: MAE = (1/n) × Σ|y_real - y_predicho|

Mide el error promedio absoluto entre las predicciones y los valores reales. Un MAE de 1.658 significa que, en promedio, las predicciones se desvían 1.658 unidades del valor real.

6.2 RMSE (Root Mean Squared Error)

Fórmula: RMSE = √[(1/n) × Σ(y_real - y_predicho)²]

Similar al MAE pero penaliza más los errores grandes. Un RMSE de 3.502 indica que los errores grandes tienen más impacto en esta métrica.

6.3 R² (Coeficiente de Determinación)

Fórmula: R² = 1 - (SS_res / SS_tot)

Indica qué proporción de la varianza en los datos es explicada por el modelo. Un R² de 0.744 significa que el modelo explica el 74.4% de la variación en la demanda.

  • R² = 1.0: Predicción perfecta
  • R² = 0.744: Nuestro modelo (bueno)
  • R² = 0.0: El modelo no explica nada

6.4 MAPE (Mean Absolute Percentage Error)

Fórmula: MAPE = (1/n) × Σ|(y_real - y_predicho)/y_real| × 100

Expresa el error como porcentaje. Un MAPE del 70.8% indica el error promedio porcentual de las predicciones. Este valor es alto porque muchos productos tienen ventas bajas o cero.

7.2 Análisis de Resultados

  • Precisión aceptable: Un R² de 0.744 es bueno para un primer modelo con datos limitados.
  • Error moderado: El MAE de 1.658 es razonable dado el rango de ventas.
  • Mejorable: El MAPE del 70% indica que hay margen de mejora, especialmente para productos con ventas bajas.

7.3 Limitaciones Identificadas

  1. Datos históricos limitados: Solo se usaron 30 días de historial para features.
  2. Productos con ventas cero: Muchos productos tienen días sin ventas, afectando las métricas.
  3. Sin variables externas: No se incluyeron factores como clima, promociones reales, etc.

8.3 Intervalos de Confianza

El sistema calcula intervalos de confianza al 95% usando la fórmula:

predicción ± z_score × residual_std × √(día)

Esto significa que los intervalos se amplían con el horizonte de predicción, siendo más estrechos en días cercanos y más amplios en días lejanos.

9.3 Conclusión

El sistema actual demuestra que es posible integrar machine learning en un sistema de gestión de inventario de forma práctica. Los modelos entrenados proporcionan predicciones razonablemente precisas que pueden ayudar a las empresas a tomar mejores decisiones de compra y reabastecimiento.

Las principales áreas de mejora son: aumentar la cantidad y calidad de los datos de entrenamiento, incorporar variables externas, y implementar reentrenamiento automático para mantener los modelos actualizados.
